# Quick Start — Dual Core Coder

## 🚀 30-Second Setup

### 1. Verify Dependencies
```bash
python check-deps.py
```
Should show: `✓ All dependencies OK`

### 2. Start Ollama (Two Terminals)
**Terminal 1:**
```bash
ollama serve
```

**Terminal 2:**
```bash
# Windows PowerShell
$env:OLLAMA_HOST="0.0.0.0:11435"
ollama serve

# Mac/Linux
OLLAMA_HOST=0.0.0.0:11435 ollama serve
```

### 3. Run Dual Core
**Windows:** Double-click `run.bat`  
**Mac/Linux:** `./run.sh`  
**Manual:** `python app.py`

## 📋 One-Time Setup (First Time)

### Step 1: Python Environment
```bash
# Verify Python 3.13+
python --version

# Create virtual environment (optional)
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\Activate.ps1

# Install dependencies
pip install -r requirements.txt
```

### Step 2: Ollama Installation & Setup
- Download from [ollama.com](https://ollama.com)
- Install and start:

**Terminal 1** (Alpha Server on port 11434):
```bash
ollama serve
```

**Terminal 2** (Beta Server on port 11435):
```bash
# Windows PowerShell
$env:OLLAMA_HOST = "0.0.0.0:11435"
ollama serve

# Mac/Linux
export OLLAMA_HOST=0.0.0.0:11435
ollama serve
```

### Step 3: Verify Installation
```bash
python check-deps.py   # Check Python & dependencies

# Check Ollama Alpha (port 11434)
curl http://localhost:11434/api/tags

# Check Ollama Beta (port 11435)
curl http://localhost:11435/api/tags
```

## 🎮 Using Dual Core

### In the App
1. **Pull Models** (optional) — Use ⚙️ MODEL SETTINGS to pull models from Ollama Hub
   - Common choices: deepseek-coder, mistral, codellama, neural-chat
   - Click "PULL NEW MODEL" or use "Quick Pull" with model name

2. **Select Models** — Drop-down menus for Alpha & Beta servers
   - Last selected models are remembered across sessions
   - "Test Connection" button verifies server availability

3. **Pick Mode** — Choose consensus strategy:
   - ⚡ **Parallel** — Both run simultaneously (fastest)
   - ⚔️ **Debate** — Both answer, then synthesized into final answer
   - ✓ **Verify** — Alpha generates, Beta reviews & improves

4. **Select Type** — Code or General question mode

5. **Enter Prompt** — Type your question in the prompt box
   - 🎤 **LISTEN** — Transcribe speech (Python 3.13+)
   - 🔊 **SPEAK** — Read output aloud

6. **ENGAGE DUAL CORE** — Click to start generation

### Output Display
- **Alpha Output** — First model's response with confidence score & time
  - Click **≣** to open in separate detachable window
  - Click **COPY** to copy to clipboard
  - Scrollable with persistent state
  
- **Beta Output** — Second model's response with confidence score & time
  - Same window features as Alpha
  
- **Confidence Scores** — Estimated quality percentage for each response
  - Based on response length, code signals, uncertainty indicators
  - Displayed as percentage bars below scores
  
- **Consensus Output** — Final merged/verified solution
  - Marked with "VERIFIED" badge
  - Generated by local Ollama model synthesizing all inputs
  - Same window/copy features as individual outputs

### Dashboard (Optional)
Click **◐ DASHBOARD** to open live metrics window:
- Real-time confidence gauges for Alpha & Beta
- Quality delta (+/- comparison)
- Token counts and generation speed
- Elapsed time tracking
- Current mode and model names

## ⚙️ Configuration

### Server Configuration
Default URLs work for local setup (edit in app to change):
- **Alpha**: `http://localhost:11434` (Primary server on port 11434)
- **Beta**: `http://localhost:11435` (Secondary server on port 11435)

### Model Selection & Management
Click **⚙️ MODEL SETTINGS** to:
- **Pull Models** — Add new models from Ollama Hub
  - Search/filter available models
  - Select target server (Alpha or Beta)
  - Quick-pull with model name + server
- **Browse Models** — View all available models with metadata
- **Per-Server Control** — Independently manage models on each server
- **Auto-Refresh** — Models load automatically on app startup

### Session Persistence
- Model selections saved in `.model_config.json`
- Window sizes & positions saved in `.window_geometry.json`
- Sash positions (pane splits) remembered between sessions
- All settings restore on app restart

## 🆘 Troubleshooting

### "No module named 'requests'"
```bash
pip install -r requirements.txt
```

### "No module named 'tkinter'"
- **Windows**: Reinstall Python, select "tcl/tk" option
- **Mac**: `brew install python-tk@3.10`
- **Linux**: `sudo apt-get install python3-tk` (Ubuntu) or equivalent

### "Server unreachable"
```bash
# Verify Ollama is running
curl http://localhost:11434/api/tags
curl http://localhost:11435/api/tags
```

### Models not showing
1. Check Ollama is running on both ports
2. Click **🔄 REFRESH** in Model Settings
3. Verify models exist: `ollama list`

### Slow generation
- Use smaller models (deepseek-coder, mistral)
- Ensure sufficient RAM (8GB+ recommended)
- Close other applications
